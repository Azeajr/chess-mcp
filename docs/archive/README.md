# Documentation archive

These records explain decisions made during earlier implementations. They are not sources of truth
for current code, contracts, commands, counts, or product status.

- `MCP_DESIGN_HISTORY.md`: durable MCP design principles retained from the original server.
- `REPERTOIRE_DESIGN_HISTORY.md`: decisions inherited from the Python-era repertoire design.
- `UI_DESIGN_HISTORY.md`: superseded UI proposal and the choices that replaced it.
- `CHAT_TOOLSET_REVIEW_HISTORY.md`: findings that motivated the canonical contract and chat work.
- `ENGINEERING_PASSES_HISTORY.md`: disposition of the former prompt collection.

For current behavior use `README.md`, `docs/ARCHITECTURE.md`, `docs/PWA_PRODUCT.md`, the generated
tool catalog, source, and tests.
