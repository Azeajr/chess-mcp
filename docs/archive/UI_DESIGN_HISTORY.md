# UI design history

Historical status: superseded proposal.

The proposal correctly emphasized a board-centered repertoire editor, offline engine support,
autosave, visual engine/repertoire comparison, and explicit preview before edits. The shipped stack
is SolidJS, Vite, chessground, custom components, and `styles.css`; chat uses OpenRouter's
OpenAI-compatible API. Current interaction behavior is documented in `docs/PWA_PRODUCT.md`.
